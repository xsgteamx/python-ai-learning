## 一、前置知识纠正

- Python 更准确地说是**动态类型、强类型语言**。
  - 动态类型：变量不需要提前声明类型，运行时可以指向不同类型的对象。
  - 强类型：不同类型之间不会随意自动混用，例如字符串和整数不能直接相加。
- CPython 执行 `.py` 文件时，会先编译为字节码，再由解释器执行。
- 语法错误、缩进错误通常在执行前即可发现；类型不匹配、变量未定义等问题，往往要运行到对应代码时才暴露。
- 工程开发中应依靠：类型注解、PyCharm 检查、静态检查工具、单元测试和代码评审来补充约束。
- 单行注释使用 `#`；三引号字符串更多用于文档字符串，不应简单等同于多行注释。
- 判断对象类型使用 `isinstance()`；`is` / `is not` 判断两个变量是否指向同一个对象，常见用法是判断 `None`。
- PEP 8 推荐使用 **4 个空格缩进**，不推荐 Tab。

## 二、四类常见容器

| 类型 | 特点 | Java 类比 |
|---|---|---|
| `list` | 有序、可变、可重复 | `ArrayList` |
| `tuple` | 有序、不可变、可重复 | 不可变 List |
| `set` | 无序、可变、元素不重复 | `HashSet` |
| `dict` | 键值对结构，键唯一 | `HashMap` |

### 记忆方式

- `list`：可变、有序、可重复
- `tuple`：不可变、有序、可重复
- `set`：无序、去重
- `dict`：`key-value` 映射

> `tuple` 读作“塔破 / 元组”，不是 `triple`。

## 三、Sequence 序列

Python 中常见的序列包括：

- `list`
- `tuple`
- `str`

序列的共同特征是：

- 有顺序
- 可以通过索引访问
- 支持切片
- 支持成员判断
- 支持获取长度

```python
list_data = [100, 200, 300, 400, 500]

print(list_data[0])   # 100
print(list_data[-1])  # 500
print(len(list_data)) # 5
```

### 正索引与负索引

- 正索引从左往右，从 `0` 开始。
- 负索引从右往左，从 `-1` 开始。
- `-1` 表示最后一个元素，`-2` 表示倒数第二个元素。
- 不从 `-0` 开始，是因为 `-0` 与 `0` 相同，会和正索引冲突。

```python
list_data = [100, 200, 300, 400, 500]

print(list_data[-1])  # 500
print(list_data[-2])  # 400
```

## 四、切片

标准格式：

```python
序列[start:end:step]
```

核心规则：**左闭右开**，包含 `start`，不包含 `end`。

```python
list_data = [100, 200, 300, 400, 500]

print(list_data[1:3])  # [200, 300]
print(list_data[2:])   # [300, 400, 500]
print(list_data[:2])   # [100, 200]
print(list_data[:])    # 复制整个列表
```

### 为什么采用左闭右开

- 切片长度可以直接计算为 `end - start`。
- 相邻切片拼接时不会重复元素。
- 与 `range()` 等 Python 规则保持一致。

### 倒序切片

```python
list_data[::-1]
```

这里的 `-1` 不是索引，而是步长，表示每次向前移动一位，因此得到反转后的新列表。

## 五、List 的创建与修改

```python
list_data = [100, 200, 300, 400, 500]
```

Python 的 `list` 可以存放不同类型的元素：

```python
mixed = [100, "ABC", True]
```

但工程开发中通常仍建议一个列表保持相同或相近的数据结构，便于理解、检查和维护。

### 修改指定元素

```python
list_data[0] = -1
```

结果：

```python
[-1, 200, 300, 400, 500]
```

### 修改切片

```python
list_data[2:4] = ["A", "B", "C"]
```

原来的 `300` 和 `400` 被替换，结果为：

```python
[100, 200, "A", "B", "C", 500]
```

切片赋值可以改变列表长度，替换元素的数量不需要与原切片长度一致。

## 六、List 的添加操作

### `append()`

把一个对象整体追加到列表末尾。

```python
list_data.append(600)
```

### `insert(index, value)`

在指定索引位置插入元素，原位置及其后面的元素整体后移。

```python
list_data.insert(2, 700)
```

结果：

```python
[100, 200, 700, 300, 400, 500]
```

底层更接近 Java 的 `ArrayList`，不是链表；中间插入需要移动后续元素。

### `extend()`

把另一个可迭代对象中的元素逐个追加到当前列表末尾。

```python
list1 = [1, 2]
list2 = ["A", "B"]

list1.extend(list2)
# [1, 2, "A", "B"]
```

### 列表加法

两个列表使用 `+` 会生成一个新列表，不会修改原列表。

```python
list3 = list1 + list2
```

区别：

- `append()`：追加一个整体对象
- `extend()`：逐个追加元素，修改原列表
- `+`：拼接后生成新列表

## 七、List 的删除操作

### `del`

删除指定索引或切片，不返回被删除的值。

```python
del list_data[2]
del list_data[1:3]
```

### `remove(value)`

删除列表中第一次出现的指定值。

```python
list_data.remove(300)
```

### `pop(index)`

删除并返回指定位置的元素；不传索引时，默认删除并返回最后一个元素。

```python
last_value = list_data.pop()
value = list_data.pop(1)
```

### `clear()`

清空列表中的所有元素。

```python
list_data.clear()
```

## 八、遍历列表

### 直接遍历元素

```python
for value in list_data:
    print(value)
```

### 使用下标遍历

```python
for index in range(len(list_data)):
    print(index, list_data[index])
```

### `enumerate()`

`enumerate` 读作“依纽默瑞特”，用于遍历时同时获得下标和元素。

```python
for index, value in enumerate(list_data):
    print(index, value)
```

## 九、嵌套列表

列表中可以继续存放列表，常用于二维表格、矩阵等结构。

```python
matrix = [
    [1, 2],
    [3, 4]
]

print(matrix[1][0])  # 3
```

嵌套本身很常见，但层级过深会降低可读性。工程中应优先使用清晰的数据结构或对象封装。

## 十、列表推导式

列表推导式用于快速完成简单的映射和过滤。

### 基本写法

```python
squares = [x ** 2 for x in range(5)]
```

结果：

```python
[0, 1, 4, 9, 16]
```

### 带条件

```python
squares = [x ** 2 for x in range(10) if x % 2 == 0]
```

实际执行顺序是：

1. 从 `range(10)` 中取出 `x`
2. 判断 `x % 2 == 0`
3. 条件成立后，再计算 `x ** 2`
4. 把结果放进新列表

最终结果：

```python
[0, 4, 16, 36, 64]
```

### 使用边界

列表推导式适合：

- 简单映射
- 简单过滤
- 一行就能看明白的逻辑

不适合：

- 多层嵌套
- 多个复杂条件
- 有副作用的操作
- 需要大量解释才能理解的代码

**可读性优先。复杂逻辑应拆成普通 `for` 循环。**

## 十一、多重列表推导式与 `zip()`

### 多重推导式

```python
result = [(i, j) for i in [1, 2] for j in ["A", "B"]]
```

等价于嵌套循环：

```python
result = []
for i in [1, 2]:
    for j in ["A", "B"]:
        result.append((i, j))
```

结果是笛卡尔积：

```python
[(1, "A"), (1, "B"), (2, "A"), (2, "B")]
```

多重列表推导式可读性较差，团队开发中应谨慎使用。

### `zip()`

如果需要一一对应配对，应使用 `zip()`：

```python
numbers = [1, 2, 3]
letters = ["A", "B", "C"]

result = list(zip(numbers, letters))
```

结果：

```python
[(1, "A"), (2, "B"), (3, "C")]
```

- `zip()` 本身返回一个 `zip` 对象。
- 转换为 `list` 后，每一项是一个 `tuple`。

## 十二、常用 List 方法与函数

| 操作 | 作用 |
|---|---|
| `append(x)` | 末尾追加一个对象 |
| `insert(i, x)` | 在指定位置插入元素 |
| `extend(iterable)` | 逐个追加可迭代对象中的元素 |
| `remove(x)` | 删除第一次出现的指定值 |
| `pop(i)` | 删除并返回指定位置元素，默认末尾 |
| `clear()` | 清空列表 |
| `index(x)` | 返回指定值第一次出现的位置 |
| `count(x)` | 统计指定值出现次数 |
| `sort()` | 原地排序 |
| `reverse()` | 原地反转当前顺序 |
| `copy()` | 浅拷贝列表 |
| `len(list_data)` | 获取元素个数 |
| `max(list_data)` | 获取最大值 |
| `min(list_data)` | 获取最小值 |
| `sum(list_data)` | 计算元素之和 |
| `list(iterable)` | 把可迭代对象转换为列表 |

### `sort()` 与 `reverse()` 的区别

- `sort()`：按照规则排序，直接修改原列表。
- `sort(reverse=True)`：降序排序。
- `reverse()`：只把当前顺序反转，不负责判断大小。

```python
numbers = [3, 1, 2]

numbers.sort()
# [1, 2, 3]

numbers.reverse()
# [3, 2, 1]
```

### `max()`、`min()`、`sum()` 的限制

- `max()`、`min()` 要求元素之间可以比较。
- `sum()` 要求元素可以进行数值加法。
- 混合不兼容类型时会报错。

## 十三、今天形成的工程认知

1. Python 的灵活性很高，但默认约束少，工程可靠性需要自己增加护栏。
2. `list` 虽然允许存不同类型，但实际项目中仍应保持结构统一。
3. 切片的左闭右开初看不直观，但规则统一、计算方便。
4. 列表推导式不是必须使用的语法糖，只适合简单映射和过滤。
5. 多重列表推导式虽然简短，但可读性差，应优先考虑普通循环。
6. 对 Java 开发者来说，可以把 `list` 理解为更灵活的 `ArrayList`，但要主动补充类型注解和测试。
