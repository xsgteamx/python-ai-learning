链表（Linked List）

定义： 链表是由一系列节点组成的线性数据结构，每个节点包含数据域和指针域，通过指针将节点串联起来。

核心特征：
分散存储：节点在内存中不一定连续
动态扩展：无需预分配空间
顺序访问：只能从头部或已知节点开始遍历
指针开销：每个节点需要存储额外指针

存储原理
内存地址（分散存储）：
节点1（0x1000）    节点2（0x2000）    节点3（0x3000）
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│ data: 10    │    │ data: 20    │    │ data: 30    │
│ next: 0x2000│───▶│ next: 0x3000│───▶│ next: NULL  │
└─────────────┘    └─────────────┘    └─────────────┘

访问 arr[2]：需要从头遍历 0x1000 → 0x2000 → 0x3000

链表变体
类型	                描述                       	特点
单向链表	    每个节点只有指向下一个的指针    	只能单向遍历
双向链表	    每个节点有 prev 和 next 指针	   双向遍历，更灵活
循环链表	    尾节点指向头节点	可循环遍历
双向循环链表	双向 + 循环	最灵活
    
核心操作特性
操作	时间复杂度	原理说明
头部插入	O(1)	直接修改头指针
头部删除	O(1)	直接修改头指针
尾部插入	O(n)	需遍历到尾节点（有尾指针 O(1)）
尾部删除	O(n)	需遍历到倒数第二个节点
中间插入	O(n)	需遍历到插入位置
中间删除	O(n)	需遍历到目标节点
随机访问	O(n)	需从头遍历
按值查找	O(n)	需遍历所有节点
反转    	O(n)	需遍历所有节点

优缺点分析
优点	缺点
插入删除高效（O(1)）	    随机访问慢（O(n)）
动态扩展，无容量限制	    额外指针占用内存
内存利用灵活	           不支持随机访问
适合频繁变化的数据	        实现复杂
无内存浪费	                缓存不友好

适用场景
推荐使用：
频繁在头部插入/删除
数据量动态变化
实现 LRU 缓存
实现栈、队列等结构
图的邻接表表示
音乐播放列表

不推荐使用：
需要频繁随机访问
内存受限的环境
需要频繁查找元素
