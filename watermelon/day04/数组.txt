数组（Array）

定义： 数组是由相同类型的元素组成的集合，这些元素存储在连续的内存空间中，通过索引（下标）进行访问。

核心特征：
连续内存分配：所有元素在内存中紧密排列
固定或动态容量：静态数组固定大小，动态数组可扩容
随机访问：通过索引可在 O(1) 时间内访问任意元素
类型一致性：所有元素类型相同（Python 列表允许不同类型）

存储原理
内存地址递增方向 →
┌─────┬─────┬─────┬─────┬─────┐
│ 0   │ 1   │ 2   │ 3   │ 4   │   ← 索引
├─────┼─────┼─────┼─────┼─────┤
│ 10  │ 20  │ 30  │ 40  │ 50  │   ← 数据
└─────┴─────┴─────┴─────┴─────┘
1000  1004  1008  1012  1016     ← 内存地址

访问 arr[3]：直接计算地址 = 1000 + 3 × 4 = 1012



核心操作特性

操作	 时间复杂度	     原理说明
随机访问	O(1)	    直接通过索引计算内存地址
顺序访问	O(n)	    需要遍历所有元素
尾部插入	O(1)均摊	直接在末尾添加，扩容时 O(n)
尾部删除	O(1)    	直接减少长度
中间插入	O(n)	    需要移动后续所有元素
中间删除	O(n)	    需要移动后续所有元素
头部插入	O(n)	    需要移动所有元素
头部删除	O(n)	    需要移动所有元素
按值查找	O(n)	    线性搜索（无序）
按值查找	O(log n)	二分查找（有序）
排序	  O(n log n)	快排/归并/Timsort

动态数组扩容机制
扩容策略：
当数组容量不足时，分配更大的内存空间（通常是原来的 2 倍）
将旧数据复制到新内存
释放旧内存
扩容的摊还分析：
单次插入最坏情况 O(n)
n 次插入总时间 O(n)
平均每次插入 O(1)

缩容策略：
当元素数量低于容量的 1/4 时，缩容为 1/2
防止内存浪费

优缺点分析
优点	                    缺点
随机访问速度快（O(1)）	     插入删除需要移动元素（O(n)）
内存连续，缓存友好	         固定大小，动态扩容有开销
实现简单直观	             内存可能浪费（预分配）
支持高效的排序和查找	      不适合频繁插入删除

适用场景
推荐使用：

需要频繁随机访问数据                                      
数据量可预估
操作主要在尾部进行
需要排序和二分查找
实现矩阵运算
数据缓存

不推荐使用：
频繁在中间插入/删除
数据量剧烈变化
需要频繁改变大小
