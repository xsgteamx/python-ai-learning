算法基础
算法是用于解决特定问题的有限指令序列（计算机可执行的操作），即解决问题的具体方法。

算法的五大特性：

特性	说明
输入	0个或多个输入
输出	至少1个输出
有穷性	有限步骤后自动结束，不会无限循环
确定性	每一步含义确定，无二义性
可行性	每一步清楚可行，能用纸笔计算求出答案



算法的分类
按应用目的分类：

--搜索算法：深度优先搜索、广度优先搜索等
--排序算法：冒泡排序、插入排序、选择排序、快速排序、归并排序等
--最优化算法等

按实现策略分类：

---暴力法、增量法、分治算法、动态规划算法、贪心算法等

时间复杂度
1）时间复杂度的定义
时间复杂度描述运行一个算法所需时间资源的多少，使用执行的基本指令数量来衡量。

常见基本指令：

--赋值指令
--算术运算指令（加、减、乘、除）
--逻辑运算指令（与、或、非）
--比较指令

假设每条基本指令执行时间为一个固定单位，算法所需时间表示为与输入规模 n 相关的函数：T(n)

三种时间复杂度：

类型	说明
最优时间复杂度	最少需要多少基本操作（价值不大）
最坏时间复杂度	最多需要多少基本操作（主要关注）
平均时间复杂度	平均需要多少基本操作（全面但有前提）
核心关注最坏时间复杂度。

2）时间复杂度的大O表示法
当 n 趋近无穷大时，忽略 T(n) 中的低阶项、常数项及高阶项的常数系数，用最高阶项表示增长趋势。

示例：

算法A：T(n) = 5n + 5 → O(n)

算法B：T(n) = 2n² + 2n - 2 → O(n²)

3）常见时间复杂度
大小关系：

text
O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(n·2ⁿ) < O(n·n!) < O(nⁿ)
注：log₂n 常简写为 log n。

常见案例：

复杂度  	典型算法    	 示例代码
O(1)	    两数求和	    return a + b
O(log n)	二分查找	    每轮缩减一半
O(n)	    数组求和	    单层循环遍历
O(n log n)	归并排序	    分治后合并
O(n²)	    冒泡排序	    双层嵌套循环
O(n·2ⁿ)	    枚举全部子集    位运算生成
O(n·n!)	    全排列	        递归排列


空间复杂度
1）空间复杂度的定义
空间复杂度描述算法运行所需内存空间随输入规模变大时的变化趋势。

内存占用内容：

--指令：编译后的二进制指令
--数据：变量、常量、对象等（主要关注）

2）常见空间复杂度
复杂度	说明	典型示例
O(1)	空间与输入规模无关	原地反转数组
O(n)	空间随输入线性增长	用集合判断重复元素
O(n²)	空间随输入平方增长	矩阵转置
循环中初始化变量或函数调用占用的内存在进入下一循环后会被释放，不会累积，空间复杂度仍为 O(1)。
