# Python + AI 学习君子协定

> 可以学得慢，可以暂时卡住，但不能悄悄放弃。

## 1. 学习周期

自双方第一次有效提交起，连续执行 **12 周**。期满后共同复盘，再决定是否开启下一期。

## 2. 每周最低要求

每人每周至少向本仓库提交 **1 次有效学习成果**，放在自己的文件夹中。

一次有效成果至少包含：

- 自己运行或修改过的 Python / AI 代码、Notebook 或练习；
- 运行结果、截图、输出文件或问题解决记录；
- 一份简短总结，说明本周学了什么、亲手做了什么、还有什么没弄懂。

不要求每天打卡，也不硬性计算学习时长，只看每周是否留下真实、可验证的学习痕迹。

## 3. 有效成果的判断

满足下面两点，才算有效：

1. **有证据**：仓库中能看到代码、记录和运行结果；
2. **能说明**：本人能用自己的话解释大致逻辑，并完成一个简单修改或重新运行。

可以使用 AI、教程和网络资料，但只有复制粘贴、无法解释或无法修改的内容，不算有效成果。

对方可以偶尔进行一次不超过 **5 分钟**的随机验收，例如：

- 解释某段代码的作用；
- 预测修改后的运行结果；
- 改一个参数、条件或输出格式；
- 说明遇到的报错以及解决过程。

验收不是考试，只用于确认本人确实参与了学习。

## 4. 请假与补交

- 遇到加班、生病、出差或其他现实事务，可以提前说明并请假；
- 每人 12 周内最多有 **2 个普通请假周**；
- 第一次未完成且未请假，可以在下一周补交；
- 补交周需要完成“上周补交 + 本周成果”。

## 5. 什么算半途而废

出现以下任意一种情况，视为半途而废：

- 未请假，连续 **2 周**没有有效成果；
- 第一次缺交后，下一周仍未完成补交；
- 明确表示不再继续；
- 多次提交无法解释的复制内容，或故意伪造学习记录。

## 6. 违约约定

谁半途而废，谁就在判定后的 **7 天内**，给坚持的一方购买一份 **《战地 6》标准版**；若无法购买，则支付双方认可的同等金额。

如果双方都违约，双方都要履行，不得以“大家一起摆烂”为理由相互免责。
